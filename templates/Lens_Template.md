# Lens Template

Use this template when adding a new custom lens to `lenses/`. Kaleidoscope will automatically load any markdown file placed there, so keeping the structure consistent makes the generated review more useful.

## Lens Name
Name the lens clearly. Example: Security Lens

## Purpose
Describe what this lens is for, what kind of concerns it surfaces, and why it adds value beyond the default lenses.

## When to Use
- Describe the situations where this lens matters most.
- Include the kinds of problems, decisions, or ambiguity this lens is especially good at clarifying.

## Key Questions
- What questions should this lens pressure-test?
- What does this perspective notice that others might miss?
- What would a strong evaluator ask before being comfortable with the decision?

## What This Lens Looks For
- Name the signals, patterns, or tensions this lens pays attention to.
- Include the kinds of tradeoffs or blind spots it tends to surface.

## Common Risks / Failure Modes
- List the mistakes, omissions, or weak assumptions this lens is designed to catch.
- Include subtle or second-order failure modes when possible.

## Signals or Evidence to Consider
- List the kinds of evidence, data, or conversations that make analysis from this lens more grounded.
- Include both quantitative and qualitative evidence when relevant.

## Output Expectations
- Explain what a strong review from this lens should contain.
- Note whether the output should prioritize risks, recommendations, questions, scenarios, or tradeoffs.

## Recommendation Style
Describe the tone and posture of the recommendations. For example: direct but conditional, scenario-based, skeptical, user-centered, or financially disciplined.
